# 배포 방법 (Docker)

## 1. .dockerignore 작성

프로젝트 루트에 `.dockerignore` 생성:

```
.git
__pycache__
*.pyc
images/
videos/
output.mp4
readme/
*.md
```

## 2. Docker Desktop Windows 컨테이너 모드 전환

Docker Desktop 기본값은 Linux 컨테이너라 DirectML이 동작하지 않습니다.
빌드 전에 반드시 Windows 컨테이너 모드로 전환하세요.

```
작업 표시줄 트레이의 Docker 아이콘 우클릭
→ "Switch to Windows containers..."
→ 확인
```

> 전환 후 기존 Linux 이미지는 사용 불가 — 새로 빌드 필요합니다.

## 3. 이미지 빌드

```bash
docker build -t yolo-hand-detection .
```

## 4. 컨테이너 실행

```bash
# 기본 실행
docker run -p 8911:8911 yolo-hand-detection

# 환경변수로 설정 변경
docker run -p 8911:8911 \
  -e YOLO_CONFIDENCE=0.5 \
  -e YOLO_THRESHOLD=0.45 \
  -e YOLO_NETWORK=v11 \
  yolo-hand-detection
```

## 5. 동작 확인

```bash
curl http://localhost:8911/api/v1/health
```

---

## 환경변수 목록

| 변수명 | 기본값 | 설명 |
|---|---|---|
| `YOLO_NETWORK` | `v11` | 사용할 모델 (`v11` 또는 darknet) |
| `YOLO_CONFIDENCE` | `0.7` | 탐지 신뢰도 임계값 |
| `YOLO_THRESHOLD` | `0.45` | NMS 임계값 |
| `YOLO_SIZE` | `640` | 입력 이미지 크기 |
| `YOLO_PORT` | `8911` | 서버 포트 |

---

## 주의사항

- Windows 컨테이너 베이스 이미지(`windowsservercore`)는 수 GB로 첫 빌드가 오래 걸립니다. 이후에는 레이어 캐시로 빠릅니다.
- Windows 컨테이너는 Windows 호스트에서만 실행 가능합니다.
- `models/` 폴더의 `.onnx`, `.weights` 파일이 이미지에 포함되어 이미지 크기가 클 수 있습니다.
